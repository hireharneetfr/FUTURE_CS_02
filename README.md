# Cyber Security Task 2 – Phishing Email Analysis

## Overview

This project was completed as part of the Future Interns Cyber Security Internship Program. The objective of the task was to analyse phishing emails, identify phishing indicators, assess potential risks, and recommend security measures to help users recognise and prevent phishing attacks.

The report examines four phishing email samples that impersonate trusted organisations and services. Each email was analysed using cybersecurity tools and techniques to determine its legitimacy and identify malicious characteristics.

## Objectives

* Identify phishing emails and common phishing techniques.
* Analyse sender information and email characteristics.
* Perform email header analysis using MXToolbox.
* Analyse suspicious domains, links, and indicators using VirusTotal.
* Identify phishing indicators and associated risks.
* Assess the potential impact of phishing attacks.
* Recommend security controls and prevention measures.

## Tools Used

### MXToolbox

MXToolbox was used to analyse email header information and investigate sender domains. The tool helps identify spoofing attempts, authentication failures, and domain-related issues that may indicate phishing activity.

Website: https://mxtoolbox.com

### VirusTotal

VirusTotal was used to analyse suspicious domains, URLs, and phishing indicators. The platform aggregates results from multiple security vendors and threat intelligence sources to identify potentially malicious content.

Website: https://www.virustotal.com

## Phishing Samples Analysed

1. DHL Shipping Invoice
2. Human Resources Survey Email
3. Outlook Account Verification Email
4. PayPal Account Restriction Email

## Analysis Approach

The phishing emails were analysed using the following methodology:

1. Collection of phishing email samples.
2. Identification of sender information and email subjects.
3. Email header examination using MXToolbox.
4. Analysis of suspicious domains and indicators using VirusTotal.
5. Identification of phishing indicators.
6. Risk assessment and threat classification.
7. Evaluation of potential security impact.
8. Development of prevention recommendations.

## Key Findings

The analysis identified several common phishing characteristics across all samples, including:

* Suspicious sender addresses.
* Brand impersonation.
* Urgent or threatening language.
* Requests for account verification.
* Unexpected attachments.
* Credential harvesting attempts.
* Malicious or suspicious links.

The phishing emails relied heavily on social engineering techniques to manipulate users into revealing sensitive information or interacting with malicious content.

## Security Recommendations

* Verify sender identities before responding.
* Avoid clicking suspicious links.
* Do not open unexpected attachments.
* Enable Multi-Factor Authentication (MFA).
* Use strong and unique passwords.
* Keep systems and software updated.
* Report suspicious emails immediately.
* Participate in cybersecurity awareness training.

## Conclusion

This project demonstrated how phishing attacks use social engineering techniques to exploit user trust and obtain sensitive information. Through the use of MXToolbox and VirusTotal, multiple phishing indicators were identified across the analysed samples. The findings highlight the importance of user awareness, security best practices, and technical controls in reducing the risk of phishing attacks. Organisations that combine employee education with strong cybersecurity measures are better positioned to defend against phishing threats and protect sensitive information.
